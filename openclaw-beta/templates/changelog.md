# Changelog Template

## Format
Based on [Keep a Changelog](https://keepachangelog.com/).

```markdown
# Changelog

## [Unreleased]

### Added
- <New feature description> (#issue-id)

### Changed
- <Changed behavior description> (#issue-id)

### Fixed
- <Bug fix description> (#issue-id)

### Removed
- <Removed feature description> (#issue-id)

### Security
- <Security fix description> (#issue-id)

## [X.Y.Z] - YYYY-MM-DD

### Added
- ...
```

## Guidelines
- Group changes by type (Added, Changed, Fixed, Removed, Security)
- Reference issue IDs for traceability
- Write for humans, not machines
- Most recent version first
- Include date for each release
